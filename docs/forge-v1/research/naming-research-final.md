# Naming Research — Final Round: Iconic Scientist/Engineer/Mathematician Names

**Date:** 2026-03-18
**Scope:** 15 names checked against npm, PyPI, GitHub CLI tools, and major brand conflicts.

---

## 1. Turing

**GitHub:** Heavy usage. `turing-machines/tpi` is an active CLI tool for Turing Pi boards. `turnetwork/turing-cli` (crypto). Multiple Turing machine simulators. Turing Machines is an active GitHub org.

**npm:** `turing` package exists (old, v0.0.74, 11 years ago). `turing-machine`, `turing-machine-js`, scoped packages `@turing-machine-js/machine` all present.

**PyPI:** `turing` package exists. `turing_machine`, `Turing-Pi-BMC`, `turing-sdk`, `turing-bot-client` all present.

**Major brand conflict:** **YES — critical.** Turing (turing.com) is a well-funded AI coding company that raised $111M Series E in March 2025 at a $2.2B valuation. It is a key coding provider for OpenAI and other LLM producers. Direct overlap with developer tooling space. This name is **completely taken** in the AI/developer tools category.

**Verdict: NOT viable.**

## 2. Babbage

**GitHub:** `openspending/babbage` (OLAP analytics engine, pip-installable), `tomscript/babbage` (data manipulation CLI), `ONSdigital/babbage` (UK government web service). `ProjectBabbage` org exists (Bitcoin/BSV ecosystem).

**npm:** `@babbage/sdk`, `@babbage/sdk-ts` (both deprecated but active Bitcoin/BSV SDKs), `babbage.ui`, `babbage-bsv`. Active `@babbage` scoped namespace.

**PyPI:** `babbage` package exists — OLAP engine for PostgreSQL with pip install.

**Major brand conflict:** Multiple Babbage AI/software companies active in 2025: Babbage Insight (data analytics), Babbage LLC (web/AI), Babbage Consulting (AI), Babbage & Turing (app dev), Babbage IT (staffing). The name is historically associated with "Boole & Babbage" (absorbed by BMC in 1999). Name is fragmented across many small players — no single dominant brand.

**Verdict: MARGINAL.** PyPI and npm both occupied. Multiple competing small brands. High friction.

## 3. Boole

**GitHub:** Low signal. `boole.nvim` (Neovim plugin for toggling booleans). Some academic/research repos. No dominant CLI tool.

**npm:** `boole` package exists — chained logical conditions library, v0.2.2, small project with 0 dependents.

**PyPI:** No direct `boole` package found. Boolean-adjacent packages (`boolean.py`, `booleano`) exist but none named exactly `boole`.

**Major brand conflict:** BooleBox / Boole Server — an Italian enterprise data security company with active product and customers in 2025. Historical association with Boole & Babbage (absorbed by BMC 1999). The "BooleBox" brand is distinct enough that confusion is possible but limited. No dominant US tech brand using the exact name "Boole".

**Verdict: MARGINAL.** npm `boole` package is taken (low-use). PyPI is clear. Boole Server/BooleBox are European enterprise products — not dev tools. Moderate friction.

## 4. Knuth

**GitHub:** `k-nuth` org (Bitcoin Cash high-performance platform, active). `Knuth Project` on GitHub. `FilipStappers/knuth` (small project). No dominant general-purpose CLI named "knuth".

**npm:** `knuth-shuffle` (popular, Fisher-Yates), `knuth-shuffle-seeded`, `@knuth/bch` (Bitcoin Cash), `knuth-morris-pratt`. No plain `knuth` package.

**PyPI:** No `knuth` package found. Related packages use full algorithm names (KnuthB shuffle, kthbuild). Plain `knuth` namespace appears **free on PyPI**.

**Major brand conflict:** KNUTH Machine Tools (German precision machinery company — completely unrelated industry). The k-nuth Bitcoin Cash project is the most significant tech use. Donald Knuth himself is a living legend — naming a tool after him carries risk of seeming presumptuous, and he is famously particular about his name/brand.

**Verdict: CAUTION.** npm `knuth` namespace appears free (only algorithm-specific packages). PyPI free. However, living person's name + Bitcoin Cash project occupies the brand. Cultural risk of naming a product after a living legend.

## Overall Findings

**None of the 15 names are cleanly available.** The landscape is fully saturated at this tier of iconic scientist names. Key patterns:

1. **Hardest conflicts (funded AI companies in the dev tools / adjacent space):**
   - Turing — $2.2B AI coding company, March 2025
   - Fermat — FERMÀT Commerce $45M Series B, AI for brands
   - Nash — Nash.ai (a16z/YC backed AI logistics)
   - Hopper — Hopper Security ($7.6M, OSS security tooling)
   - Faraday — faraday.ai + Faraday Future (public company)

2. **Taken on PyPI + npm with active maintenance:**
   - Planck (physics engine npm, planck-repl CLI)
   - Volta (volta-cli JS toolchain)
   - Dirac (CERN scientific framework on PyPI)
   - Kelvin (Kelvin Inc SDK on PyPI, active 2025)
   - Gauss (Samsung Gauss AI product)

3. **Least conflicted (still marginal):**
   - **Boole** — npm `boole` package is tiny and unmaintained; PyPI is clear; brand conflict is European enterprise security (BooleBox), not developer tools. Most defensible of the 15 but still carries friction.
   - **Knuth** — npm/PyPI namespaces largely free; but naming a product after a living, famously particular computer science legend carries cultural/reputational risk. The k-nuth Bitcoin platform occupies the GitHub org.

4. **Recommendation:** If forced to choose from this list, **Boole** is the least conflicted from a developer tooling perspective — but even it requires accepting the `boole` npm package exists and the BooleBox enterprise brand. No name on this list offers a clean, uncontested namespace.

## Deviation Analysis

### Patterns (grouped by root cause)

| Root Cause | Count | Deviations | Pattern |
|-----------|-------|-----------|---------|
| **Missing enforcement gates** | 4 | #4, #9, #10, #11 | Skills instructed behavior but didn't enforce it. Lead skipped composing-teams, TDD, and user confirmation because nothing blocked progression. Fix pattern: add HARD-GATEs. |
| **Inherited upstream debt** | 2 | #1, #2 | Path conventions and directory structure inherited from superpowers. Fix pattern: audit and restructure. |
| **Ceremony doesn't scale** | 2 | #5, #6 | Full two-stage review and individual commits were overkill for mechanical tasks. Fix pattern: add tiers (review tiers, squash-on-merge). |
| **Platform limitations** | 2 | #7, #8 | Task cleanup and status visibility limited by Claude Code's team coordination. Fix pattern: add explicit cleanup routines and text-based status reporting. |
| **Dogfooding special case** | 1 | #3 | Worktree isolation unnecessary when developing on own repo with no external users. Fix pattern: deferred as cosmetic — addressed by execution ergonomics. |

**Key insight:** 4 of 11 deviations (36%) were caused by missing enforcement. The single most impactful improvement pattern is: **if a step must not be skipped, add a HARD-GATE.** This was applied successfully in v0.2.1 (composing-teams) and v0.3.1 (user confirmation).

## Cross-Dimension Patterns

1. **"Enforcement > instruction"** — The most effective fix pattern across the entire Beta was adding HARD-GATEs. Instructions get skipped; gates don't. This should be the default approach for any critical workflow step going forward.

2. **"Planning skills work, execution skills are untested"** — Forge's design→plan→execute pipeline is well-exercised for the planning half but the execution/verification half has never run on real code. v0.4.0 is the critical test.

3. **"Patch versions are effective"** — The v0.X.1 pattern (ship fixes fast between feature releases) worked well. Deviations logged during a feature version get fixed immediately in a patch, and the fixes are available for the next feature version.

4. **"Dogfooding finds real issues but has coverage gaps"** — Building Forge with Forge found 11 real deviations and all were actionable. But documentation-only work doesn't exercise TDD, verification, or code review — the v0.4.0 reference projects are essential to cover those gaps.
